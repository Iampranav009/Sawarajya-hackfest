About the Event

Swarajya Hack Fest is a hybrid coding hackathon organized by AWS Cloud Club – JDIET in collaboration with the Department of Computer Science, JDIET. The event aims to encourage students to build innovative, scalable, cloud-based solutions using modern technologies and AWS services — promoting problem-solving, innovation, teamwork, and real-world application development.

Rules & Eligibility

🎓 Open to currently enrolled UG (Undergraduate) students only. PG / working professionals are not eligible.
📋 Must build a solution for a problem statement from the official provided list. Custom problem statements are not accepted.
🚀 No mandatory tech stack! Build with anything — React, Python, Node, Flutter, whatever you love! AWS services usage is encouraged and will earn bonus points during evaluation.
  • Teams of 2–3 members (no solo participation)
  • All code written during the 24-hour window only
  • Public GitHub repository required
  • AI tools allowed ✅ (GitHub Copilot, AWS Kiro, etc.)
  • No plagiarism — immediate disqualification
  • Must attend at least one mentorship session
  • AWS services usage gives evaluation priority
  • Submission requires: PPT (official template) + Data Report + GitHub repo URL + Deployment URL (optional) — all via the Project Submission Portal
  • Valid student ID / proof of enrollment required at registration

Organized by AWS Cloud Club – JDIET

AWS Cloud Club – JDIET is a student-led technical initiative at Jawaharlal Darda Institute of Engineering and Technology, focused on building awareness and hands-on experience in cloud computing and modern technologies. The club regularly conducts workshops, hackathons, technical sessions, and cloud learning initiatives to help students develop practical skills for the modern tech industry.

Frequently Asked Questions

Who can participate?▾
Open to currently enrolled UG (Undergraduate) students only. PG students and working professionals are not eligible.
What is the team size?▾
Teams must have 2 to 3 members. Solo participation is not allowed.
Is there a registration fee?▾
The 1st round (online) is free. For the shortlisted teams, the 2nd round (offline) has a registration fee of ₹300.
Do we have to use a specific tech stack?▾
No! You can use any language, framework, or tool you prefer. However, using AWS services will be given priority during evaluation — so it's encouraged but not mandatory.
Will there be mentorship?▾
Yes! Two structured mentorship sessions will be conducted during Round 1.
How is Round 1 conducted?▾
Round 1 begins with an online kickoff on Google Meet, followed by a continuous 24-hour coding window starting 15 March 2026.
How are teams shortlisted?▾
Based on innovation, technical quality, AWS service usage, and overall feasibility of the solution.
What do we need to submit?▾
Your Round 1 submission must include: (1) PPT Presentation using the official template (PPT/PPTX format, max 15 MB); (2) Data Report — a written description of your project, the problem chosen, your solution approach, and expected impact; (3) Public GitHub Repository URL with all source code; and (4) Deployment/Live Demo URL (optional but strongly recommended). All submissions are made via the Project Submission Portal.
How do we submit our project?▾
Go to the Project Submission Portal on this website. Only the Team Leader should submit the form on behalf of the entire team. Fill in your team details, paste your GitHub and deployment URLs, describe your project in the Data Report field, and upload your PPT file using the official template. Download the template from the Submission section on this page.

How to Submit Your Project

Round 1 submission is done entirely online via the Project Submission Portal

📊
1. PPT Presentation

Prepare your presentation using the official PPT template. It must cover the problem, solution, implementation approach, tech stack, and expected impact. Only PPT/PPTX format, max 15 MB.

📝
2. Data Report

A written project description entered in the submission form — include the problem statement chosen, your solution approach, tech stack used, and expected real-world impact.

💻
3. GitHub Repository

Submit the URL of your public GitHub repository containing all source code written during the 24-hour hackathon window. Private repos will not be accepted.

🌐
4. Deployment URL (Optional)

If your project is hosted (Vercel, Netlify, AWS, etc.), paste the live demo link. Deployed projects score higher during evaluation.

ℹ️
Submission Process

Only the Team Leader submits the form on behalf of the entire team. Download the official PPT template, complete all slides, then open the Project Submission Portal and fill in your team details, paste your links, describe your project, and upload the PPT file. Each team submits only once.
