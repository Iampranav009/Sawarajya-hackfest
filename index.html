About the Event

Swarajya Hack Fest is a hybrid coding hackathon organized by AWS Cloud Club – JDIET in collaboration with the Department of Computer Science, JDIET. The event aims to encourage students to build innovative, scalable, cloud-based solutions using modern technologies and AWS services — promoting problem-solving, innovation, teamwork, and real-world application development.

Rules & Eligibility

🎓 Open to currently enrolled UG (Undergraduate) students only. PG / working professionals are not eligible.
🚀 No mandatory tech stack! Build with anything — React, Python, Node, Flutter, whatever you love! AWS services usage is encouraged and will earn bonus points during evaluation.
  • Teams of 2–3 members (no solo participation)
  • All code written during the 24-hour window only
  • Public GitHub repository required
  • AI tools allowed ✅ (GitHub Copilot, AWS Kiro, etc.)
  • No plagiarism — immediate disqualification
  • Must attend at least one mentorship session
  • AWS services usage gives evaluation priority
  • Submission: GitHub link + deployment URL + description via Google Form

Organized by AWS Cloud Club – JDIET

AWS Cloud Club – JDIET is a student-led technical initiative at Jawaharlal Darda Institute of Engineering and Technology, focused on building awareness and hands-on experience in cloud computing and modern technologies. The club regularly conducts workshops, hackathons, technical sessions, and cloud learning initiatives to help students develop practical skills for the modern tech industry.

Frequently Asked Questions

Who can participate?▾
Open to currently enrolled UG (Undergraduate) students only. PG students and working professionals are not eligible.
What is the team size?▾
Teams must have 2 to 3 members. Solo participation is not allowed.
Is there a registration fee?▾
No! Registration is completely free of charge.
Do we have to use a specific tech stack?▾
No! You can use any language, framework, or tool you prefer. However, using AWS services will be given priority during evaluation — so it's encouraged but not mandatory.
Will there be mentorship?▾
Yes! Two structured mentorship sessions will be conducted during Round 1.
How is Round 1 conducted?▾
Round 1 begins with an online kickoff on Google Meet, followed by a continuous 24-hour coding window starting 15 March 2026.
How are teams shortlisted?▾
Based on innovation, technical quality, AWS service usage, and overall feasibility of the solution.
What do we need to submit?▾
Public GitHub repo link + deployment URL (if available) + project description via Google Form.
